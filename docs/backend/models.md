# Database Models

We use **SQLAlchemy 2.0+** with the declarative mapping style for our database models.

## Core Models

Defined in `app/db/models.py`.

### `User`
- **Table**: `users`
- **Purpose**: Represents a user of the system.
- **Fields**: `id` (primary key), `created_at`.

### `Thread`
- **Table**: `threads`
- **Purpose**: Represents a single chat conversation.
- **Fields**: `id`, `owner_id`, `title`, `created_at`.
- **Relationships**: Belongs to a `User`.

### `ThreadMessage`
- **Table**: `thread_messages`
- **Purpose**: Stores individual messages within a thread.
- **Fields**: `id`, `thread_id`, `owner_id`, `role` (user/assistant), `content`, `status`, `run_id`, `created_at`, `updated_at`.

### `ThreadRun`
- **Table**: `thread_runs`
- **Purpose**: Tracks a single execution flow of an agent.
- **Fields**: `id`, `thread_id`, `status`, `input_message`, `selected_file_ids`, `workspace_files`, `output_text`, `error_detail`, `created_at`, `started_at`, `completed_at`.

### `ThreadRunEvent`
- **Table**: `thread_run_events`
- **Purpose**: High-granularity log of agent steps and tool calls.
- **Fields**: `id`, `run_id`, `thread_id`, `sequence`, `event_type`, `name`, `node_name`, `payload` (JSON), `created_at`.

### `ThreadFile`
- **Table**: `thread_files`
- **Purpose**: Metadata for files uploaded by the user or generated by the agent.
- **Fields**: `id`, `thread_id`, `object_key`, `filename`, `kind` (upload/artifact), `content_type`, `size`, `status`, `created_at`.

### `ThreadSandboxSession`
- **Table**: `thread_sandbox_sessions`
- **Purpose**: Maps a chat thread to a specific environment in the `microsandbox-executor`.
- **Fields**: `thread_id`, `sandbox_session_id`, `executor_base_url`, `created_at`, `updated_at`.
- **Lifecycle Note**: The backend consults this mapping during thread deletion so the executor can delete the live session before the database record is removed.

## Base Configuration

All models inherit from a common `Base` class, which uses SQLAlchemy's `DeclarativeBase`. We use type hints (`Mapped`) for all column definitions to ensure type safety.
